# 衛生学公衆衛生学II クイズ・ビンゴ

添付PowerPointの24問を使った、スマートフォン参加型のリアルタイム・ビンゴです。

## できること

- 学生はQRコードまたは6桁のルームコードから参加
- アプリのインストール・学生アカウント登録は不要
- 参加者ごとに24問の配置が異なる5×5カードを自動作成
- 教員画面から出題、10秒カウント、正解発表、次の問題へ進行
- 正解したマスを自動で開放
- リーチ人数、回答人数、正答率を教員画面に表示
- 1列または2列ビンゴを選択可能
- 先着3名を自動判定し、教員・学生画面に表示
- ニックネームのみを保存。作成から2日を過ぎた古いルームは、新規ルーム作成時に削除

## フォルダ構成

- `public/`：学生・教員が見る画面
- `functions/api.js`：回答、ビンゴ判定、順位判定を行うCloudflare Pages Function
- `schema.sql`：D1データベース作成用
- `wrangler.example.toml`：Wranglerを使う場合の見本
- `当日用カンペ.txt`：授業当日の進行例

## Cloudflare Pagesへの設置

### 1. GitHubへアップロード

このフォルダの中身を、新しいGitHubリポジトリへそのままアップロードします。

重要：ZIPファイル自体ではなく、`public`、`functions`、`schema.sql`などがリポジトリ直下に見える状態にしてください。

### 2. D1データベースを作成

Cloudflareの管理画面でD1データベースを1つ作成します。

推奨名：`public-health-bingo`

作成したデータベースのコンソールで、`schema.sql`の内容をすべて貼り付けて実行します。

### 3. Pagesプロジェクトを作成

Cloudflare PagesからGitHubリポジトリを接続します。

- Framework preset：None
- Build command：空欄
- Build output directory：`public`

### 4. D1をPagesへ接続

Pagesプロジェクトの設定にあるBindingsで、D1 database bindingを追加します。

- Variable name：`DB`　※必ず大文字
- D1 database：先ほど作成した `public-health-bingo`

保存後、Pagesを再デプロイします。

### 5. 動作確認

公開URLを開いて、次の順番で確認します。

1. 「新しいルームを作る」
2. 別のスマホでQRコードを読み込む
3. ニックネームを入力して参加
4. 教員画面で「ゲーム開始」
5. 学生が回答
6. 「正解発表」
7. 正解者のマスが緑色になる

## 当日の推奨設定

試験後5〜10分なら、勝利条件は **1列ビンゴ** を推奨します。

40人が全問正解すると仮定した自動テストでは、ランダム配置によりおおむね10問前後で上位3名が決まりました。学生の誤答が入るため、実際には10〜15問程度が目安です。

2列ビンゴは時間が長くなりやすいため、時間に余裕がある場合に使ってください。

## 注意点

- 本名ではなくニックネーム参加を推奨します。
- 順位はサーバーが正解処理を受け付けた順です。通信差があるため、最終的には学生のビンゴ画面も確認してください。
- QR画像は外部のQR生成サービスを利用します。QRが表示されない場合も、画面の6桁ルームコードを学生が手入力すれば参加できます。
- PowerPointに記載された設問・選択肢を基にしています。統計値を次年度以降も使う場合は、元教材側の更新に合わせて `functions/api.js` と `public/app.js` の問題データも更新してください。

## 出典となったPowerPointの扱い

問題文・選択肢は、提供された「ビンゴボード_選択肢改行付き - 修復済み.pptx」の内容をアプリ化しています。日本の総人口、出生率、粗死亡率などの数値も、外部データで置き換えず、PowerPoint記載の選択肢を維持しています。
